## Database Details

**Database:** medicvista_retailer (LOCAL)
**Table:** retailer_pending_status_updates

**Columns:**
- id (BIGINT AUTO_INCREMENT)
- request_id (BIGINT) - wholesaler request ID
- new_status (VARCHAR(30)) - PENDING/PROCESSING/COMPLETED/FAILED
- queued_at (DATETIME) - when queued

## Why medicvista_retailer?

The pending status updates are part of the retailer sync system's offline queue.
This is LOCAL retailer data, NOT Django ERP data.

- `medicvista_retailer` → Retailer app's own database
- `pharma_db` → Django ERP database (only for retailer_request table)
